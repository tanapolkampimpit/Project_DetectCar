2.1.2 config.py
ตำแหน่ง:
  app/projects/detect_car/config.py

หน้าที่:
  - เป็นไฟล์ตั้งค่าหลักของ detect_car
  - กำหนด path ของ model weight ทั้ง 3 ตัว
  - กำหนดค่า runtime เช่น device, batch size, queue depth, file size limit, blur threshold, confidence threshold
  - อ่าน environment variable ด้วย prefix DETECT_CAR_

ค่าหลักที่กำหนด:
  - environment: ค่า environment ของ detect_car
  - use_gpu: เปิด/ปิด GPU
  - device: cpu หรือ cuda
  - model_path: path ของ ConvNeXt weight
  - yolo_gen_path: path ของ YOLO photo type classifier
  - yolo_damage_path: path ของ YOLO damage detector
  - batch_max_size: จำนวน request สูงสุดต่อ batch
  - batch_max_wait: เวลารอรวม batch สูงสุด
  - num_batch_workers: จำนวน worker สำหรับ batch inference
  - max_queue_depth: จำนวน request สูงสุดในระบบก่อน reject
  - max_file_bytes: ขนาดไฟล์สูงสุด default 25MB
  - blur_threshold: ค่า threshold สำหรับตรวจภาพเบลอ default 50.0
  - min_car_area_ratio: สัดส่วนพื้นที่รถขั้นต่ำ ใช้ดูว่ารถไกลเกินไปหรือไม่
  - damage_confidence_threshold: confidence ขั้นต่ำของ damage detection
  - damage_nms_iou_threshold: ค่า IoU สำหรับกรอง box ซ้อนกัน
  - damage_image_size: image size สำหรับ YOLO damage
  - match_threshold: confidence ขั้นต่ำสำหรับถือว่า match กับ expected_view

function สำคัญ:
  - normalize_runtime(logger=None)
    ตรวจว่า GPU ใช้งานได้จริงหรือไม่ หากตั้งค่าให้ใช้ CUDA แต่เครื่องไม่มี CUDA จะ fallback เป็น CPU

2.1.3 lifecycle.py
ตำแหน่ง:
  app/projects/detect_car/lifecycle.py

หน้าที่:
  - เป็น lifecycle controller ของ detect_car router
  - ทำงานตอน startup และ shutdown ของ router
  - โหลดโมเดลทั้งหมด
  - สร้าง BatchInferenceEngine
  - เริ่ม worker สำหรับ inference
  - cleanup resource ตอนปิดระบบ

ลำดับการทำงานตอน startup:
  1. เรียก settings.normalize_runtime()
  2. โหลดโมเดลด้วย load_models()
  3. ดึง event loop ปัจจุบัน
  4. สร้าง BatchInferenceEngine โดยส่ง convnext, yolo_gen, yolo_damage เข้าไป
  5. เรียก engine.start()
  6. เก็บ engine ไว้ที่ app.state.detect_car_engine
  7. ถ้าใช้ GPU จะสร้าง task สำหรับเคลียร์ VRAM เมื่อ memory reserved สูง

ลำดับการทำงานตอน shutdown:
  1. เรียก engine.stop()
  2. cancel cleanup task ถ้ามี
  3. เคลียร์ CUDA cache ถ้าใช้ GPU

endpoint ที่มี:

2.2.1 POST /predict_view
หน้าที่:
  - ตรวจสอบว่ารูปที่ส่งมาเป็นมุมหรือประเภทภาพที่ตรงกับ expected_view หรือไม่
  - ใช้สำหรับงานตรวจรูปถ่ายรถ เช่น ลูกค้าส่งรูป Front แต่ระบบต้องตรวจว่ารูปนั้นเป็น Front จริงหรือไม่

input:
  - file: UploadFile
  - expected_view: Form string default = Front

flow:
  1. สร้าง request_id เพื่อใช้ trace log/response
  2. validate expected_view ด้วย validate_expected_view()
  3. validate content_type ว่าต้องเป็น image/*
  4. ขอ slot จาก backpressure gate
  5. อ่านไฟล์ด้วย read_file_safe()
  6. preprocess ภาพด้วย prepare_image_for_inference()
  7. ถ้าภาพ invalid จะ return invalid_image_format
  8. ถ้า blur_score ต่ำกว่า threshold จะ return image_too_blurry
  9. สร้าง BatchItem ด้วย prepare_single_item()
  10. submit item เข้า BatchInferenceEngine
  11. await future เพื่อรอผล inference
  12. เอา key damages ออก เพราะ endpoint นี้โฟกัส view prediction
  13. เพิ่ม request_id แล้ว return response

2.2.2 POST /predict_damage
หน้าที่:
  - ตรวจจับรอยความเสียหายจากรูปภาพ
  - คืนรายการ damage พร้อม confidence และ bounding box

input:
  - file: UploadFile

flow:
  1. สร้าง request_id
  2. validate content_type ว่าต้องเป็น image/*
  3. ขอ slot จาก backpressure gate
  4. อ่านไฟล์ด้วย read_file_safe()
  5. decode และวิเคราะห์ blur ด้วย decode_and_analyze_image()
  6. ถ้าภาพ invalid จะ return invalid_image_format
  7. ถ้าภาพเบลอ จะ return image_too_blurry พร้อม damages=[]
  8. สร้าง item แบบ SimpleNamespace
  9. เรียก predict_damage_sync() ใน threadpool
  10. เพิ่ม request_id แล้ว return response
  11. release backpressure ใน finally

2.3.3 core/engine.py
ตำแหน่ง:
  app/projects/detect_car/core/engine.py

หน้าที่:
  - เป็น inference engine หลักของระบบ
  - จัดคิว request
  - รวมหลาย request เป็น batch
  - เรียกใช้โมเดล YOLO general, ConvNeXt และ YOLO damage
  - รวมผลลัพธ์กลับไปยัง request แต่ละตัว
  - เก็บ metrics สำหรับ endpoint /status

class สำคัญ:

BatchItem:
  - dataclass สำหรับเก็บข้อมูล request ที่รอ inference
  - field หลัก:
    - request_id
    - expected_view
    - img
    - tensor
    - future
    - blur_score
    - enqueue_at

BackpressureGate:
  - ใช้จำกัดจำนวนงาน active ในระบบ
  - ป้องกัน queue เต็มหรือ server รับงานมากเกินไป
  - method:
    - acquire(): ขอ slot ก่อนรับงาน
    - release(): คืน slot หลังงานเสร็จ
    - current_count: จำนวนงาน active ปัจจุบัน

TelemetryTracker:
  - เก็บสถิติของ inference
  - metric ที่เก็บ:
    - total_processed
    - total_errors
    - latency_sum
    - batch_sizes
    - uptime
  - ใช้ใน /status endpoint

BatchInferenceEngine:
  - engine หลักที่ถือ model ทั้ง 3 ตัว
  - field สำคัญ:
    - convnext
    - yolo_gen
    - yolo_damage
    - queue
    - yolo_gen_lock
    - yolo_damage_lock

method สำคัญ:
  - start()
    เริ่ม worker ตาม NUM_BATCH_WORKERS
  - stop()
    หยุด worker และคืน exception ให้ request ที่ยังค้างใน queue
  - submit_nowait(item)
    ส่ง BatchItem เข้า queue
  - _worker(worker_id)
    ดึง item จาก queue แล้วรวมเป็น batch ตาม BATCH_MAX_SIZE หรือ BATCH_MAX_WAIT
  - _run_batch_sync(batch)
    ทำ inference จริงใน thread pool

ลำดับ inference ใน _run_batch_sync:
  1. filter request ที่ถูก cancel แล้ว
  2. รวม tensor ด้วย torch.stack()
  3. ย้าย tensor ไปยัง device cpu/cuda
  4. เรียก YOLO general เป็น gatekeeper เพื่อตรวจประเภทภาพ
  5. เลือกภาพที่ควรตรวจ damage
  6. เรียก YOLO damage เฉพาะภาพที่เข้าเงื่อนไข
  7. เลือกภาพที่ควรเข้า ConvNeXt เพื่อจำแนกมุมรถภายนอก
  8. เรียก ConvNeXt แล้ว softmax เป็น probability
  9. map output index ของ ConvNeXt ให้ตรงกับ label ของระบบ
  10. เรียก build_result() เพื่อรวมผล
  11. set_result กลับไปที่ future ของแต่ละ request
  12. record metrics
  13. release backpressure

2.4.2 models/convnext.py
ตำแหน่ง:
  app/projects/detect_car/models/convnext.py

class:
  - MultiTaskConvNeXt

หน้าที่:
  - นิยามโมเดล ConvNeXt สำหรับจำแนกมุมรถภายนอก
  - ใช้ timm.create_model("convnext_small", pretrained=False, num_classes=1024)
  - ต่อด้วย class_prediction = nn.Linear(1024, 8)

output:
  - โมเดลเดิมทำนาย 8 class สำหรับมุมรถภายนอก
  - มีการเพิ่ม roof_logits เป็นค่า -100 เพื่อให้ output รวมเป็น 9 logits สำหรับ compatibility
  - Roof จะไม่ถูกทำนายจาก ConvNeXt เพราะ probability แทบเป็น 0

class order ใน comment ของ code:
  - class_prediction order:
    - 0: Front
    - 1: Front-Left
    - 2: Left
    - 3: Back-Left
    - 4: Back
    - 5: Back-Right
    - 6: Right
    - 7: Front-Right

ลำดับ output ที่ engine คาดหวัง:
  - 0: Front
  - 1: Back
  - 2: Left
  - 3: Right
  - 4: Roof
  - 5: Front Right
  - 6: Front Left
  - 7: Back Right
  - 8: Back Left

3. API Endpoint
router.py สร้าง APIRouter พร้อม lifespan ของ detect_car โดย endpoint ภายใน router มี 3 รายการหลัก:

3.1 POST /predict_view
หน้าที่:
  - รับรูปภาพ 1 ไฟล์ และ expected_view
  - ตรวจสอบว่ารูปเป็น image/*
  - จำกัดขนาดไฟล์ไม่เกิน DETECT_CAR_MAX_FILE_BYTES หรือค่า default 25MB
  - ตรวจ blur score ถ้าต่ำกว่า threshold จะคืน error image_too_blurry
  - ส่งภาพเข้า BatchInferenceEngine เพื่อทำนายมุมภาพ/ประเภทภาพ
  - ตัดข้อมูล damages ออกจาก response เพราะ endpoint นี้เน้นผล view prediction

Input:
  - file: UploadFile แบบ multipart/form-data
  - expected_view: Form string ค่า default คือ Front

expected_view ที่รองรับ:
  - Front
  - Front-Left
  - Left
  - Back-Left
  - Back
  - Back-Right
  - Right
  - Front-Right
  - Roof
  - Interior
  - SpareTire
  - ChassisNumber
  - Accessories
  - Dashcam
  - Odometer
  - TaxSticker
  - RegistrationDoc
  - EngineCompartment
  - TireFrontLeft
  - TireFrontRight
  - TireBackLeft
  - TireBackRight
  - Others

Response สำเร็จ:
  - prediction.label
  - prediction.confidence
  - is_car
  - match
  - quality.is_blurry
  - quality.blur_score
  - quality.is_too_far
  - quality.car_area_ratio
  - time_ms
  - request_id

Error ที่พบได้:
  - 400 invalid_expected_view
  - 400 invalid_file_type
  - 400 invalid_image_format
  - 413 file_too_large
  - 429 queue_full
  - 500 internal_server_error
  - response body แบบ status=error เมื่อภาพเบลอเกิน threshold

3.2 POST /predict_damage
หน้าที่:
  - รับรูปภาพ 1 ไฟล์

4. Function Code
ส่วนนี้สรุป function code แบบเน้นให้คนที่มารับช่วงต่อสามารถแก้ไขโค้ดได้ง่ายขึ้น โดยอธิบายว่า function แต่ละตัวอยู่ไฟล์ไหน ทำหน้าที่อะไร รับข้อมูลอะไร คืนค่าอะไร และถ้าต้องการแก้ behavior บางอย่างควรเริ่มจากไฟล์ใด

4.1 ภาพรวม Flow ของ Function Code
ระบบ detect_car มี flow หลัก 2 เส้น คือ predict_view และ predict_damage

4.1.1 Flow: predict_view
ใช้ตรวจว่ารูปที่อัปโหลดมาเป็นประเภทรูปหรือมุมรถที่ตรงกับ expected_view หรือไม่

ลำดับการทำงาน:
  1. router.py รับ request ที่ /predict_view
  2. validate expected_view ด้วย services/view_service.py
  3. ตรวจว่าไฟล์เป็น image ด้วย router.py
  4. ตรวจ queue capacity ด้วย core/engine.py
  5. อ่านไฟล์ upload ด้วย read_file_safe()
  6. decode รูป + คำนวณ blur + preprocess tensor ด้วย services/preprocessor.py
  7. ถ้ารูปเบลอเกิน threshold จะ return error ทันที
  8. สร้าง BatchItem ด้วย prepare_single_item()
  9. ส่ง BatchItem เข้า BatchInferenceEngine
  10. engine.py รวม request เป็น batch
  11. YOLO photo type classifier ตรวจประเภทภาพก่อน
  12. ถ้าเป็น exterior/others จะเรียก ConvNeXt เพื่อแยกมุมรถ
  13. ถ้าเข้าเงื่อนไข จะเรียก YOLO damage เพื่อดึง damage เพิ่มภายใน engine
  14. analyzer.py รวมผล model ทั้งหมด
  15. router.py ตัด damages ออกจาก response ของ predict_view
  16. return prediction, is_car, match, quality, time_ms, request_id

แก้ไขง่าย:
  - อยากเพิ่ม expected_view ใหม่: แก้ core/labels.py
  - อยากเปลี่ยน threshold รูปเบลอ: แก้ config.py หรือ env DETECT_CAR_BLUR_THRESHOLD
  - อยากเปลี่ยน logic ว่ารูป match หรือไม่: แก้ services/view_matcher.py
  - อยากเปลี่ยน response field: แก้ schemas/prediction.py และ services/analyzer.py
  - อยากเปลี่ยนขั้นตอน inference: แก้ core/engine.py

4.1.2 Flow: predict_damage
ใช้ตรวจจับรอยความเสียหายของรถจากรูปภาพ

ลำดับการทำงาน:
  1. router.py รับ request ที่ /predict_damage
  2. ตรวจว่าไฟล์เป็น image
  3. ตรวจ queue capacity ด้วย backpressure
  4. อ่านไฟล์ upload
  5. decode รูปและคำนวณ blur score
  6. ถ้ารูปเบลอเกิน threshold จะ return error พร้อม damages=[]
  7. เรียก services/damage_service.py
  8. damage_service.py เรียก YOLO damage model
  9. services/detections.py extract label, confidence, bounding box
  10. ทำ NMS เพื่อลด box ซ้อนกัน
  11. return damages, quality, time_ms, request_id

แก้ไขง่าย:
  - อยากปรับ confidence ของ damage: แก้ config.py หรือ env DETECT_CAR_DAMAGE_CONFIDENCE_THRESHOLD
  - อยากปรับ NMS: แก้ config.py หรือ env DETECT_CAR_DAMAGE_NMS_IOU_THRESHOLD
  - อยากเปลี่ยนรูปแบบ box/result: แก้ services/detections.py และ schemas/prediction.py
  - อยากเปลี่ยน model damage: เปลี่ยนไฟล์ weight หรือ path ใน config.py

4.2 Router Function Code
ไฟล์:
  app/projects/detect_car/router.py

หน้าที่ของไฟล์:
  - เป็น API layer ของ detect_car
  - รับ request จาก client
  - validate input เบื้องต้น
  - ส่งงานไปยัง service หรือ inference engine
  - จัดการ error response ให้เป็นรูปแบบเดียวกัน

4.2.1 raise_api_error(status_code, error, message, **extra)
หน้าที่:
  - สร้าง HTTPException สำหรับ API error
  - รวม error, message และข้อมูลเสริม เช่น request_id ไว้ใน detail

Input:
  - status_code: HTTP status code เช่น 400, 413, 429, 500
  - error: error code แบบ string เช่น invalid_file_type
  - message: ข้อความอธิบาย error
  - extra: field เพิ่มเติม เช่น request_id, valid_values

Output:
  - ไม่มี return เพราะ raise HTTPException

ใช้เมื่อ:
  - request ไม่ถูกต้อง
  - ไฟล์ใหญ่เกิน
  - queue เต็ม
  - เกิด internal server error

ถ้าจะแก้:

4.2.4 predict_view(request, file, expected_view)
หน้าที่:
  - endpoint หลักสำหรับตรวจประเภทรูปและมุมรถ
  - ตรวจว่าภาพที่ส่งมาตรงกับ expected_view หรือไม่

Input:
  - request: FastAPI Request ใช้ดึง app.state.detect_car_engine
  - file: UploadFile แบบ multipart/form-data
  - expected_view: Form string default = Front

Output:
  - PredictViewResponse หรือ ImageQualityError

ขั้นตอนสำคัญใน code:
  - สร้าง request_id ด้วย uuid.uuid4().hex[:8]
  - validate expected_view ด้วย validate_expected_view()
  - validate image content type
  - acquire backpressure
  - read file
  - prepare_image_for_inference()
  - check blur_score
  - สร้าง BatchItem
  - submit เข้า engine queue
  - await item.future
  - result.pop("damages", None)
  - ใส่ request_id แล้ว return

จุดที่คนแก้ต่อควรรู้:
  - predict_view ไม่ได้เรียก model ตรงๆ แต่ส่งเข้า BatchInferenceEngine
  - response ของ engine มี damages แต่ endpoint นี้ลบออกก่อน return
  - ถ้าอยากให้ /predict_view คืน damages ด้วย ให้เอาบรรทัด result.pop("damages", None) ออก และปรับ schema PredictViewResponse
  - ถ้า queue เต็มจะคืน 429 queue_full
  - finally จะ release backpressure เฉพาะกรณี submit ไม่สำเร็จ เพราะถ้า submit สำเร็จ engine จะ release ให้เอง

4.2.5 predict_damage(request, file)
หน้าที่:
  - endpoint หลักสำหรับตรวจจับ damage

  - file: UploadFile

  - ใส่ request_id แล้ว return
  - release backpressure ใน finally เสมอ

จุดที่คนแก้ต่อควรรู้:
  - endpoint นี้ใช้เฉพาะ YOLO damage model
  - ถ้าต้องการให้ตรวจเฉพาะภาพรถจริงก่อน ต้องเพิ่ม YOLO general gatekeeper ก่อนเรียก predict_damage_sync

4.5.2 BackpressureGate
หน้าที่:
  - จำกัดจำนวน request ที่ระบบรับพร้อมกัน
  - ป้องกัน memory/queue overload

method:
  - acquire()
    ถ้า current_count น้อยกว่า max จะเพิ่ม count และ return True
    ถ้าเต็มแล้ว return False

  - release()
    ลด current_count ลง แต่ไม่ให้ต่ำกว่า 0

  - current_count
    คืนจำนวน active task ปัจจุบัน

ถ้าจะแก้:
  - ถ้าต้องการเปลี่ยน capacity ให้แก้ DETECT_CAR_MAX_QUEUE_DEPTH
  - ถ้าต้องการแยก limit ระหว่าง predict_view กับ predict_damage ต้องสร้าง gate แยกหรือเพิ่ม logic ใน router.py

หน้าที่:
  - เก็บ performance metrics ของ batch inference

method:
  - record_batch(size, avg_latency_ms, error_count)
    บันทึกจำนวน request, latency เฉลี่ย และ error ใน batch

  - get_stats()
    คืน total_processed, total_errors, avg_latency_ms, avg_batch_efficiency, uptime_seconds

ถ้าจะแก้:
  - ถ้าต้องการเพิ่ม p95 latency ต้องเก็บ latency list เพิ่มใน class นี้
  - ถ้าเพิ่ม metric ต้องแก้ schemas/health.py และ router.py ด้วย

4.5.4 BatchInferenceEngine
หน้าที่:
  - เป็นตัวควบคุม inference หลักของ predict_view
  - เก็บโมเดล 3 ตัว คือ convnext, yolo_gen, yolo_damage
  - จัด queue, worker, batch และ threadpool inference

method สำคัญ:

start()
  - สร้าง worker task ตาม settings.NUM_BATCH_WORKERS
  - worker แต่ละตัวจะรออ่าน item จาก queue

stop()
  - cancel worker ทั้งหมด
  - ถ้ายังมี item ค้างใน queue จะ set exception กลับไปที่ future
  - release backpressure ของ item ที่ค้าง

submit_nowait(item)
  - ใส่ BatchItem เข้า asyncio.Queue
  - ถ้า queue เต็มจะ raise asyncio.QueueFull

_worker(worker_id)
  - ดึง item แรกจาก queue
  - รอ item เพิ่มจนกว่าจะครบ BATCH_MAX_SIZE หรือเกิน BATCH_MAX_WAIT
  - ส่ง batch ไป run ใน MODEL_THREAD_POOL ผ่าน _run_batch_sync()

_run_batch_sync(batch)
  - function ที่มี logic inference หลัก
  - ทำงานใน threadpool เพื่อไม่ block event loop

ลำดับ logic ใน _run_batch_sync:
  1. ลบ item ที่ future ถูก cancel แล้ว
  2. รวม tensor เป็น batch ด้วย torch.stack()
  3. ย้าย tensor ไปยัง settings.DEVICE
  4. ถ้าใช้ GPU และโมเดลเป็น float16 จะ cast tensor เป็น half
  5. แปลง img RGB เป็น BGR สำหรับ YOLO
  6. เรียก yolo_gen เพื่อจำแนกประเภทภาพ
  7. เลือก damage_indices จากผล yolo_gen
  8. เรียก yolo_damage เฉพาะภาพที่ต้องตรวจ damage
  9. เลือก cnn_indices สำหรับภาพที่ควรเข้า ConvNeXt
  10. เรียก convnext เฉพาะ cnn_indices
  11. softmax logits เป็น probability
  12. map probability index ให้ตรง label ของระบบ
  13. เรียก build_result() ทีละ item
  14. set_result หรือ set_exception กลับไปที่ future
  15. record metrics
  16. release backpressure และ clear img/tensor

จุดที่คนแก้ต่อควรรู้:
  - yolo_gen และ yolo_damage มี lock เพราะ Ultralytics YOLO อาจไม่ thread-safe เมื่อถูกเรียกพร้อมกัน
  - ConvNeXt ใช้ batch tensor แต่ YOLO ใช้ list ของ image
  - ถ้าเพิ่มโมเดลใหม่ ควรเพิ่มใน load_models(), lifecycle.py และ constructor ของ BatchInferenceEngine
  - ถ้าปรับ label order ของ ConvNeXt ต้องแก้ mapping ใน _run_batch_sync ด้วย
  - อย่าลืม release backpressure เมื่อเพิ่ม error path ใหม่

4.7 Service Function Code

4.7.1 services/preprocessor.py
หน้าที่:
  - เตรียมรูปก่อนเข้า model

decode_and_analyze_image(content)
  - Input: bytes ของรูป
  - Process:
    1. cv2.imdecode bytes เป็น BGR image
    2. ถ้า decode ไม่ได้ return None
    3. แปลงเป็น grayscale
    4. คำนวณ blur_score ด้วย cv2.Laplacian(...).var()
    5. แปลง BGR เป็น RGB
  - Output: tuple(img_rgb, blur_score) หรือ None

prepare_image_for_inference(content)
  - Input: bytes ของรูป
  - Process:
    1. เรียก decode_and_analyze_image()
    2. ถ้า decode ไม่ได้ return None
    3. preprocess รูปเป็น tensor
    4. ถ้าใช้ GPU จะพยายาม pin_memory()
  - Output: tuple(img, blur_score, tensor) หรือ None

ถ้าจะแก้:
  - ถ้าต้องเปลี่ยนขนาด input ของ ConvNeXt ให้แก้ Resize((224, 224))
  - ถ้าต้องเปลี่ยน normalization ให้แก้ Normalize(...)
  - ถ้าต้องเพิ่ม quality check เช่น overexposure/dark image ให้เพิ่มใน decode_and_analyze_image() และ schema/response

4.7.2 services/model_loader.py
หน้าที่:
  - โหลดโมเดลทั้งหมดที่ระบบต้องใช้

load_models()
  - Input: ไม่มี ใช้ settings จาก config.py
  - Output: tuple(convnext, yolo_gen, yolo_damage)
  - Process:
    1. สร้าง MultiTaskConvNeXt()
    2. โหลด weight จาก settings.MODEL_PATH
    3. แก้ key checkpoint legacy ถ้ามี fc_projection
    4. load_state_dict(strict=False)
    5. eval()
    6. ถ้า USE_GPU จะใช้ half precision
    7. โหลด YOLO general จาก settings.YOLO_GEN_PATH
    8. โหลด YOLO damage จาก settings.YOLO_DAMAGE_PATH
    9. warmup model ด้วย dummy input

ถ้าจะแก้:
  - ถ้าเปลี่ยนชื่อไฟล์โมเดล ให้แก้ config.py หรือ env path
  - ถ้าเพิ่มโมเดลใหม่ ให้ return model เพิ่มและปรับ lifecycle.py/engine.py
  - ถ้า checkpoint structure เปลี่ยน ให้แก้ logic map key ก่อน load_state_dict()

4.7.3 services/analyzer.py
หน้าที่:
  - รวมผลจาก YOLO general, ConvNeXt และ YOLO damage เป็น response สุดท้าย

build_result(item, probs, yolo_gen_out, yolo_damage_out, total_ms, settings)
  - Input:
    - item: BatchItem
    - probs: probability จาก ConvNeXt
    - yolo_gen_out: output จาก YOLO general
    - yolo_damage_out: output จาก YOLO damage
    - total_ms: เวลาประมวลผล
    - settings: config
  - Output:
    - dict ที่มี prediction, is_car, match, quality, damages, time_ms
  - Process:
    1. extract damage detections
    2. extract yolo detections
    3. คำนวณ largest car area ratio
    4. คำนวณ is_too_far
    5. filter hallucinated interior
    6. เลือกว่าจะใช้ decide_yolo_class หรือ decide_exterior_class
    7. ถ้าไม่ใช่รถ กำหนด label เป็น Unknown และลบ damages
    8. คืน response dict

process_batch_results(results, expected_views, files)
  - ใช้แปลงผลหลายไฟล์เป็น summary list
  - ปัจจุบันยังไม่มี endpoint ที่เรียกใช้โดยตรง

ถ้าจะแก้:
  - ถ้าต้องเพิ่ม field ใน response เช่น reason, all_candidates ให้เพิ่มที่ build_result()
  - ถ้าต้องเปลี่ยนเงื่อนไข is_car หรือ match ให้ดูร่วมกับ view_matcher.py
  - ถ้าต้องเปลี่ยน rule รถอยู่ไกลเกินไป ให้แก้ min_car_area_ratio ใน config.py หรือ logic is_too_far ที่นี่

has_yolo_exterior(detections, probs, match_threshold)
  - ตรวจว่าผล YOLO ควรถูกมองว่าเป็น exterior หรือไม่
  - ถ้า top detection เป็น exterior และ confidence >= threshold จะ return True
  - ถ้า top detection เป็น others แต่ ConvNeXt confidence สูง จะ return True

decide_yolo_class(expected_view, detections, match_threshold)
  - ใช้กับ expected_view ที่ไม่ใช่มุม exterior
  - หา detection ที่ตรงกับ expected_view ผ่าน INSURANCE_ANGLE_MAP
  - ถ้ามี detection ที่ confidence >= threshold จะ return expected_view, is_car=True, match=True
  - ถ้าไม่ match จะ return best detection ที่ map ไป frontend key

build_exterior_results(probs)
  - แปลง probability จาก ConvNeXt เป็น label/confidence
  - apply SWAP_MAP เพื่อแก้ซ้าย/ขวา
  - sort จาก confidence มากไปน้อย

decide_exterior_class(expected_view, probs, detections, yolo_exterior, match_threshold)
  - ใช้สำหรับตัดสินมุมรถภายนอก
  - ถ้า YOLO ไม่ใช่ exterior จะใช้ best YOLO detection เป็นผล
  - ถ้า YOLO เป็น exterior จะใช้ ConvNeXt แยกมุม
  - คำนวณ match จาก expected_view และ confidence

ถ้าจะแก้:
  - ถ้า logic match ผิดบ่อย ให้เริ่มแก้ที่ไฟล์นี้
  - ถ้าไม่ต้องการ swap ซ้าย/ขวา ให้แก้หรือเอา SWAP_MAP ออกใน build_exterior_results()
  - ถ้าต้องการปรับ threshold เฉพาะบาง class ให้เพิ่ม rule ใน decide_yolo_class() หรือ decide_exterior_class()

4.7.6 services/detections.py
หน้าที่:
  - จัดการ output จาก YOLO model
  - แปลง output เป็น dict ที่ระบบใช้งานต่อได้

_box_iou(box_a, box_b)
  - Input: box 2 กล่องในรูปแบบ [x1, y1, x2, y2]
  - Output: IoU float
  - ใช้สำหรับ NMS ของ damage

extract_yolo_detections(yolo_gen_out)
  - Input: output จาก YOLO general
  - Output: list ของ dict {label, confidence}
  - รองรับ output 2 แบบ:
    - boxes สำหรับ detection model
    - probs สำหรับ classification model
  - แปลง label ภาษาไทยเป็นอังกฤษด้วย THAI_TO_EN_CLASS_MAP

suppress_overlapping_damages(damages, iou_threshold=0.85)
  - Input: list damages
  - Output: list damages หลังกรอง box ซ้อน
  - เรียงจาก confidence สูงไปต่ำ แล้วเก็บ box ที่ไม่ทับกับ box ที่เก็บแล้วเกิน threshold

extract_damage_detections(item, yolo_damage_out, min_confidence=0.15, nms_iou_threshold=0.85)
  - Input: output จาก YOLO damage
  - Output: list ของ damage dict {label, confidence, box}
  - filter damage ที่ confidence ต่ำกว่า min_confidence
  - round confidence และ box
  - ทำ NMS ด้วย suppress_overlapping_damages()

largest_car_area_ratio(yolo_gen_out)
  - Input: output จาก YOLO general
  - Output: float สัดส่วนพื้นที่รถที่ใหญ่ที่สุด
  - ใช้ class exterior, car, roof
  - ใช้ xyxyn ซึ่งเป็น normalized box

filter_hallucinated_interior(detections, largest_ratio, expected_view)
  - Input: detections, largest_ratio, expected_view
  - Output: detections หลัง filter
  - ถ้าภาพมีรถภายนอกใหญ่พอ จะกรอง interior/dashcam/mileage_screen ที่อาจเป็น false positive

ถ้าจะแก้:
  - ถ้า YOLO output format เปลี่ยน ให้แก้ extract_yolo_detections() หรือ extract_damage_detections()
  - ถ้า damage ซ้อนกันมาก/น้อยเกินไป ให้ปรับ nms_iou_threshold
  - ถ้า is_too_far ไม่แม่น ให้ดู largest_car_area_ratio() และ settings.MIN_CAR_AREA_RATIO

4.7.7 services/damage_service.py
หน้าที่:

predict_damage_sync(engine, item)

4.8 Model Function Code

4.8.1 models/convnext.py
MultiTaskConvNeXt
  - เป็น torch.nn.Module
  - backbone ใช้ timm.create_model("convnext_small", pretrained=False, num_classes=1024)
  - class_prediction ใช้ nn.Linear(1024, 8)
  - forward(x) คืน logits 9 ช่อง โดยเพิ่ม Roof เป็น -100

จุดที่คนแก้ต่อควรรู้:
  - ถ้า train model ใหม่ที่ output class order เปลี่ยน ต้องแก้ทั้งไฟล์นี้และ mapping ใน engine.py
  - ถ้าอยากให้ ConvNeXt ทำนาย Roof จริง ต้องเอา roof_logits mock ออกและ train head ใหม่
  - ถ้าเปลี่ยน architecture เช่น convnext_tiny/convnext_base ต้องแก้ create_model และ checkpoint loading ใน model_loader.py

4.8.2 models/yolo.py
get_yolo_model(model_path, device)
  - Input:
    - model_path: path ของ .pt
    - device: cpu หรือ cuda
  - Output:
    - YOLO model object
  - Process:
    1. ลด log ของ ultralytics เป็น ERROR
    2. โหลดโมเดลด้วย YOLO(model_path)
    3. ถ้า device เป็น cuda ให้ yolo.to(device)
    4. return yolo

ถ้าจะแก้:
  - ถ้าใช้ device แบบ cuda:0/cuda:1 ต้องปรับเงื่อนไขจาก device == "cuda" เป็น device.startswith("cuda")
  - ถ้าต้อง config YOLO เพิ่ม เช่น fuse model หรือ set overrides ให้เพิ่มใน function นี้

4.10 ข้อควรระวังเวลาแก้ Function Code
  - predict_view ใช้ BatchInferenceEngine และ future ดังนั้นต้องระวัง async/thread interaction
  - backpressure ต้อง release เสมอ ไม่อย่างนั้น active_tasks จะค้างและระบบจะเริ่มตอบ queue_full
  - YOLO model ถูกเรียกภายใต้ lock เพื่อกันปัญหา thread safety
  - ConvNeXt output มีการ map index ใหม่ใน engine.py ถ้าแก้ model ต้องเช็ก mapping นี้ด้วย
  - predict_view ลบ damages ออกจาก response ก่อน return ถ้าจะคืน damage ต้องปรับทั้ง router และ schema
  - threshold หลายตัวควรแก้ผ่าน config/env ก่อน ไม่จำเป็นต้อง hard-code ใน logic
  - labels.py เป็นจุดกลางของชื่อ class ควรแก้ที่นี่ก่อนกระจายไปแก้หลายไฟล์
  - ถ้าเพิ่ม field ใน response ต้องแก้ Pydantic schema ให้ตรงด้วย ไม่อย่างนั้น FastAPI อาจ exclude หรือ validate ไม่ผ่าน
  - ถ้าเพิ่ม endpoint ที่ใช้ model หนัก ควรพิจารณา queue/backpressure ด้วย เพื่อไม่ให้ server overload

5.2 Result

Predict View Result:

  - damages[].label: ประเภทความเสียหายตาม class ของ YOLO damage
  - damages[].confidence: confidence เป็นเปอร์เซ็นต์
  - damages[].box: bounding box ตำแหน่งความเสียหาย [x1, y1, x2, y2]
  - quality.is_blurry
  - quality.blur_score
  - time_ms
  - request_id

Status Result:
  - performance: latency, batch efficiency, total processed, error rate, uptime
  - load: queue depth, active tasks, max capacity
  - config: device, batch max, workers

5.3 Summarie
ภาพรวมการทำงานของ detect_car:

  1. API รับภาพจากผู้ใช้ผ่าน multipart/form-data
  2. ตรวจชนิดไฟล์และขนาดไฟล์
  3. Decode ภาพด้วย OpenCV
  4. คำนวณ blur score ด้วย Variance of Laplacian
  5. ถ้าภาพเบลอเกิน threshold จะคืน error ทันที
  6. สำหรับ predict_view จะ preprocess เป็น tensor 224x224 แล้วส่งเข้า batch queue
  7. BatchInferenceEngine รวมหลาย request เป็น batch เพื่อลด overhead
  8. YOLO general ตรวจประเภทภาพก่อน
  9. ถ้าเป็น exterior/others จะใช้ ConvNeXt จำแนกมุมรถ
  10. ถ้าเหมาะสมจะใช้ YOLO damage ตรวจจับความเสียหาย
  11. analyzer รวมผลเป็น response พร้อม match, quality และ confidence
  12. status endpoint ใช้ดูสุขภาพระบบและประสิทธิภาพการประมวลผล

จุดเด่น:
  - รองรับ batching และ queue เพื่อให้รับ request ได้มากขึ้น
  - มี backpressure ป้องกัน server overload
  - มี metrics สำหรับ monitoring
  - แยก logic เป็น router, services, core, models, schemas ค่อนข้างชัดเจน
  - รองรับทั้ง CPU และ GPU โดยมี fallback เมื่อ GPU ใช้งานไม่ได้

ข้อจำกัดจาก source code:
  - ไม่มี train script อยู่ใน detect_car
  - ไม่มี dataset อยู่ใน detect_car
  - ไม่มี unit test สำหรับ detect_car ในโฟลเดอร์นี้
  - ConvNeXt ไม่ได้ทำนาย Roof โดยตรง แต่ใช้ YOLO สำหรับ roof
